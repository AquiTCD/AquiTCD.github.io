# Intro Portfolio サイト仕様書

## 目的
本サイトは「六角橋アキ / AquiTCD」の名刺代わりとなるポートフォリオサイトである。
最も注力している個人開発プロジェクト「ヒトツヨ」をメインに据えつつ、その他の活動（Twitch配信、Discordコミュニティ、X）へのリンクハブとして機能させる。

**ゴール**
- ペルソナ5風のソリッドで力強い（ババーンとした）デザインによって、訪問者に強い印象を与える。
- X（Twitter）等でシェアされた際に、OGP画像単体で「何をやっている人か」「どこにアクセスすればよいか（QRコード）」が完結して伝わるようにする。

**やらないこと**
- グラスモーフィズムや一般的なフラットデザインの採用（ペルソナ5風の赤・黒・斜めを徹底する）。
- 静的なOGP画像の用意（Satoriを用いた動的生成により、情報の更新性を保つ）。

## デザイン要件・トーン＆マナー
- **テーマカラー**: 赤（`#e60012`）、黒（`#0a0a0a`）、白（`#ffffff`）を基調とする。
- **テイスト**: ペルソナ5風。ソリッドシャドウ（ぼかしのない影）、斜めのボックス（`skew(-10deg)`）、ハーフトーンドット（`radial-gradient`）を活用。
- **背景**: 旧サイトから引き継いだ「ロライマ山（テーブルマウンテン）」の画像を背景に敷き、オーバーレイでドットパターンを重ねる。
- **文字の可読性**: ボックスを `skew` で斜めにする際、内部のテキストまで歪んで読みにくくなるのを防ぐため、コンテナ全体を `skew` するのではなく `::before` 疑似要素を用いて背景のみを斜めにする実装を標準とする。

## 画面構成・コンテンツ要件
トップページ（`index.astro`）はシングルページ構成とし、以下のセクションを持つ。

### 1. プロフィール (Header)
- **名前**: 六角橋アキ / AquiTCD
- **Bio**: Xのプロフィールに準拠（「アキです。以前、世界をフラフラしてました。今はなんとなく一生懸命に生きてます…」）。
- **Philosophy（エンジニアとしての自己紹介）**: 
  「魔法使いWizardや忍者Ninjaにはなれないかもしれないけど…」から始まる、ものづくりに対する熱意とスタンスを記載。

### 2. メイン活動 (Main Activities)
優先度の高い順に上から配置する。

1. **ヒトツヨ (最重要)**
   - 格ゲー上達支援Webアプリ。
   - 紹介文、サイトURL、YouTube解説動画URL、スクリーンショット（`public/screenshot.png`）を配置。
2. **Twitch Streaming**
   - 自作AI相棒システム「パルセラ」と一緒に配信している旨を記載。
3. **社会人協力ゲーム組合 (Discord)**
   - 協力プレイしたい社会人ゲーマー向けのサーバー。
   - リンク先: `https://discord.gg/hduVYVbj3b`
4. **X (Twitter)**
   - リンク先: `https://x.com/AquiTCD`

## OGP 生成要件 (`og.png.ts`)
Xのタイムライン上で最も目を引く「リンク集」として機能させるため、動的に画像を生成する。

- **技術スタック**: Astroエンドポイント + `satori` + `@resvg/resvg-js` + `satori-html`
- **画像サイズ**: 1200 x 630 px
- **フォント**: Noto Sans JP (Google Fonts CDNから動的取得)
- **QRコード**: `api.qrserver.com` を利用して各サービスのURLをQR化し、OGP内に埋め込む。

### OGP レイアウト構成
テキストが傾かないよう、Satoriの制約（疑似要素不可、z-index非対応）を回避して `position: absolute` で背景ブロックのみを `skew` する。

- **左カラム**: メイン開発「ヒトツヨ」を大きくアピールし、専用の大きなQRコードを配置。
- **右カラム**: Twitch、Discord、X の3つを横長カードとして縦に並べ、それぞれの右端に専用のQRコードを配置。

## 制約・今後の課題
- `og.png.ts` はエッジ環境でビルドされる場合、フォント取得（`fetch`）のレイテンシやタイムアウトに注意する必要がある。
- Satoriの仕様上、すべての要素に対して `display: flex` または `display: none` を明示的に指定する必要がある。
